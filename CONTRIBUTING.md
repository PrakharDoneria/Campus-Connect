# Contributing to Campus Connect

Thank you for your interest in contributing to Campus Connect! We're excited to build this platform with the community.

## How to Contribute

We welcome contributions of all kinds, from bug fixes and feature development to documentation improvements.

### Getting Started

1.  **Fork the repository**: Create your own copy of the project to work on.
2.  **Create a branch**: Make a new branch for your changes (`git checkout -b feature/your-feature-name`).
3.  **Make your changes**: Write your code and tests.
4.  **Commit your changes**: Follow the [Conventional Commits](https://www.conventionalcommits.org/) specification.
5.  **Push to your branch**: `git push origin feature/your-feature-name`.
6.  **Open a Pull Request**: Submit a PR to the `main` branch of the original repository.

### Code Style

- We use Prettier for code formatting (run `npm run format`).
- We follow standard TypeScript and React best practices.
- Use the provided `shadcn/ui` components for building the UI.
- Write clear, commented, and modular code.

### Pull Request Process

1.  Ensure your PR is focused on a single issue or feature.
2.  Provide a clear description of the changes in your PR.
3.  Link any relevant issues.
4.  Make sure your code passes all linting and build checks.

We appreciate your help in making Campus Connect a great tool for students everywhere!
